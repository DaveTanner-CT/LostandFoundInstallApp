    Before you start

    Set up your own Lost & Found

    This guide walks you through creating your own private, working copy of the Lost & Found web app. You'll create a spreadsheet and a folder, paste in two code files that we customize for you, authorize the app, and publish it.

    Your information stays on your device This page has no server and makes no network calls. The email addresses, IDs, and settings you type are only ever inserted into the code you copy — they are never sent anywhere.

    What you'll need:

    • A Google account — ideally your school/work Google Workspace account, since it owns the data.
    • About 15 minutes.
    • The list of staff email addresses who should run the office portal (you can edit this later).
    Step 1 of 7

    Create your Google Sheet

    This spreadsheet stores every lost and found record. You don't need to add any tabs or headings — the app builds those for you later.

    1. Open a new sheet: go to sheets.new in your browser, or Drive → New → Google Sheets.
    2. Give it a clear name, e.g. Lost & Found Data.
    3. Look at the address bar. The ID is the long code between /d/ and /edit. Copy it (or copy the whole URL and paste it below — we'll pull the ID out).
    Example: 1MA9EMuPPLTfoo...FthexG
    Please paste your spreadsheet ID or URL.
    Step 2 of 7

    Create a folder for photos

    When someone adds a photo of an item, the app saves it to a folder in your Google Drive. Create that folder now.

    1. In Google Drive, click New → Folder. Name it e.g. Lost & Found Photos.
    2. Open the folder. The ID is the code after /folders/ in the address bar. Copy it (or paste the whole URL below).
    Good to know Each uploaded photo is set to "anyone with the link can view" so it can display in the public catalog. Keep this folder for photos only.
    Please paste your folder ID or URL.

    Step 4 of 7

    Create the project & paste your code

    1. Go to script.google.com and click New project.
    2. Rename it (top-left) to something like Lost & Found.
    3. Add the two files below. Each Copy button gives you the version already filled in with your settings.
    Code.gsscript
    The editor already contains a file called Code.gs. Click it, select all (Ctrl/Cmd + A), delete, then paste.
    Index.htmlHTML
    Click the + next to "Files" → HTML. Name it exactly Index (the editor adds .html for you). Delete the default contents and paste. Then save everything with Ctrl/Cmd + S.

    Step 5 of 7

    Run setup & authorize

    Running the setup function does two things at once: it builds the two tabs your spreadsheet needs, and it triggers Google's one-time permission screen. Because your Sheet ID is already baked into the code you pasted, setup knows exactly which spreadsheet to build.

    1. In the Apps Script editor toolbar, open the function dropdown (it may say doGet) and choose setupSheet.
    2. Click Run.
    3. A window says "Authorization required." Click Review permissions and pick your Google account.
    ⚠ You'll see "Google hasn't verified this app" — this is expected This warning appears for every personal script, because you built it yourself rather than publishing it to the world. Click Advanced, then "Go to [your project name] (unsafe)." It's your own code running in your own account.
    What it will ask permission for You'll be asked to let the app see and edit your Google Sheets, see, edit, create and delete your Google Drive files, and see your email address. The broad Drive wording looks alarming but is normal here: the app needs it to save item photos into your folder and to know who's signed in. Click Allow.

    Back in the editor, the run finishes. Open your spreadsheet — you should now see two tabs, Items and LostItems, each with column headings. That confirms the code and your Sheet ID are wired up correctly.

    Step 6 of 7

    Publish as a web app

    Now you'll turn the project into a link people can open.

    1. Click Deploy (top-right) → New deployment.
    2. Click the gear icon → choose Web app.
    3. Set the two options below exactly as shown.
    Execute as: User accessing the web app  (required)
    This lets the app see who is signed in, so it can show office staff their admin tools and let school users report items. If you pick "Me" instead, everyone looks anonymous and the roles break.

    Who has access:

    1. Click Deploy. If asked to authorize again, follow the same steps as before.
    2. Copy the Web app URL it gives you. That's your live app — open it yourself first to check it loads.
    Step 7 of 7

    You're live

    Your Lost & Found is running as your own private copy.

    Paste it here to keep it handy, then bookmark it. (This just stores it on screen — it isn't sent anywhere.)

    Before you share it:

    • Open the URL yourself and confirm the catalog loads.
    • Sign in as an admin email and check the Admin Portal appears.
    • Add one test found item, confirm a photo uploads, then delete the test.
    Changing the app later When you edit the code, republish with Deploy → Manage deployments → pencil icon → Version: New version → Deploy. This keeps the same URL. Creating a brand-new deployment instead would give you a different link.
